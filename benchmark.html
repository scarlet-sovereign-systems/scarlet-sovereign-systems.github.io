Rapu vs grep — the honest version

The whole truth, both where Rapu wins and where grep does. Run on public repos so you can check it yourself. The source is private; the numbers aren't.

1 · Relevance — Rapu wins, always

grep matches the literal string you typed. Rapu ranks by meaning, and surfaces the definition over the usages.

redis · query: "hash table resize"
  grep :  1 hit  (a code comment)
  rapu :  #1 → src/dict.c:1677 — the actual resize logic, ranked (score 19.91)

scarlet engine · query: "ternary matrix multiply"
  grep :  0 hits  (the literal phrase isn't in the code)
  rapu :  #1 → "TritCore GEMV Kernels — Zero-multiply ternary matrix-vector product"

For a human — or an AI agent — that doesn't know the exact identifier, this is the difference between guessing and knowing.

2 · Speed — it depends, and we'll tell you exactly how

Why: grep re-scans the whole tree every query — including binaries and build junk. Rapu pays once to build an index, then skips all of that. On a clean little tree with a couple of searches, grep's raw scan is genuinely hard to beat. On a real, messy repo you search again and again — exactly what an AI agent does — Rapu pulls ahead and stays there.

3 · Verdict

Use Rapu when you need to find code by meaning, or you're searching a big/messy repo over and over. Use grep when you already know the exact string in a small, clean tree.

Rapu is as smart as it is honest — it doesn't claim to beat grep at everything. It claims to read your code, and to win the workload that matters: an agent, searching a real repo, by intent.

Reproducible: rapu index <dir> then rapu search <query> vs grep -rin, best-of-3 warm, on public repos (redis). Measured 2026-07-02.
